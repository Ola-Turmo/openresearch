# Mode Recipes

These recipes synthesize the two source repositories into Codex-only GPT Image 2 prompting guidance. Use them as modular add-ons, not as mandatory boilerplate.

## Everyday photo

Use for realistic, non-studio, phone-like images.

Add only when compatible:

- natural light from the location
- imperfect candid framing
- phone-photo realism, slight motion blur or softness
- real-world background that matches the subject
- natural skin texture, fabric wrinkles, ordinary lens behavior

Avoid: studio lighting, airbrushed skin, over-sharpening, artificial background, excessive glare.

## Cinematic still

Use for premium film-frame composition.

Add only when compatible:

- movie-still composition
- motivated natural or practical lighting
- subtle film grain
- controlled color grade
- warm rim light or cool ambient shadows
- shallow depth of field when the subject should separate from background
- lively moment, not a static mannequin pose

Avoid: over-saturated colors, excessive vignette, plastic skin, posterized CGI unless requested.

## Product/e-commerce

Use for marketplace, catalog, product hero, product board, food/product renders.

High-leverage fields:

- product identity and locked geometry
- camera angle: front hero, three-quarter, macro detail, top-down flat lay
- surface and props: marble, podium, matte background, lifestyle desk, ingredient props
- lighting: soft diffused studio, glossy highlights, realistic reflections
- packaging text: exact labels only when user requests them
- ecommerce board layout: hero image, detail panels, feature callouts, use cases, storyboard strip

Avoid: extra products, unrelated logos, fake labels, unreadable text, distorted packaging, clutter.

## Ad creative

Use for campaign visuals, social ads, launch posters, brand systems.

High-leverage fields:

- campaign objective and audience mood
- hero product/person/mascot
- headline and CTA exact text
- typography style and placement
- brand palette and visual metaphor
- clean hierarchy: headline, hero, support copy, feature icons

Avoid: extra copy, watermark, unrelated brand marks, unreadable typography, too many competing focal points.

## Poster/illustration

Use for travel posters, editorial illustration, event art, book/album-style covers.

High-leverage fields:

- era/medium: screen print, pencil illustration, ink drawing, painterly, vector poster
- composition motif: negative space, framing objects, flowing shapes, centered emblem
- typography: title placement, date/location, slogan
- texture: paper grain, halftone, hand-drawn contour, print registration

Avoid: generic AI-gloss, clutter, illegible title text, mismatched style eras.

## UI/social mockup

Use for app screens, landing pages, dashboards, livestream overlays, social posts.

High-leverage fields:

- device or canvas ratio
- exact UI sections: header, hero, cards, chart area, nav, sidebar, chat, product card
- visual hierarchy and spacing
- typography system and icon style
- placeholder data vs exact labels
- dark/light mode and accent color

Avoid: broken alignment, impossible UI, random text, distorted icons, inconsistent spacing.

## Infographic/diagram

Use for technical explanations, process flows, visual breakdowns, maps, system diagrams.

Add:

- clear visual flow
- labeled steps or numbered panels
- simple icons and arrows
- enough technical detail to understand the relationship
- consistent line weight and spacing

Avoid: decorative clutter, unlabeled arrows, unreadable microtext, inconsistent diagram logic.

## Character/brand board

Use for mascots, turnaround sheets, expression sheets, merch boards, identity systems.

High-leverage fields:

- character core design and personality
- front/side/back views or expression count
- brand palette and motifs
- merchandise/mockup list
- organized grid with labeled sections
- consistency of character across panels

Avoid: changing character identity across panels, random merchandise, mismatched palette.

## Comparison/meme

Use for side-by-side, before/after, meme rows, quick visual jokes.

High-leverage fields:

- exact panel count and orientation
- short readable captions
- consistent subject across panels
- clear contrast between states
- casual phone/candid quality for memes when appropriate

Avoid: too many captions, unreadable small text, changing the subject between panels unless requested.
